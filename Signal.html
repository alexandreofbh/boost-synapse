Boost Synapse

Signal

In Boost Synapse different signals are identified by different user-defined pointer-to-function types. The return type of such signal typedefs is used to identify the signal. This is possible because the emit interface ignores the return value of any connected function object. The arity and the argument types of a signal typedef are used to automatically bind the correct emit overload during overload resolution.

For example, the following typedef defines a mouse_move signal that takes two integers. The mouse_move_ struct is not a "real" type: its only purpose is to identify the signal and so it's declared inline and left incomplete:

typedef struct mouse_move_(*mouse_move)( int x, int y );

Of course the typedef itself—in this case mouse_move, not mouse_move_—is used to refer to the defined signal.
