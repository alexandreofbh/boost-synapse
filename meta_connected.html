Boost Synapse

meta::connected

#include <boost/synapse/connect.hpp>

namespace
boost
    {
    namespace
    synapse
        {
        namespace
        meta
            {
            namespace
            connect_flags
                {
                unsigned const connecting=1;
                unsigned const first_for_this_emitter=2;
                unsigned const last_for_this_emitter=4;
                }
            
            template <class Signal>
            struct
            connected
                {
                typedef connected<Signal>(*type)( connection & c, unsigned flags );
                };
            }
        }
    }

This template defines a signal that is emitted by the meta::emitter every time a connection object is created (or destroyed) connecting (or disconnecting) the specified Signal. This meta signal can be used to integrate Boost Synapse with a 3rd-party callback/signal API. For an example see "Using Meta Signals to Connect to a C-Style Callback API".

Connected handlers must take two arguments: c is the connection being created or destroyed, and flags is a combination of the bits defined in the connect_flags namespace:

  • the connecting bit is set if the connection object is being created, and cleared if it's being destroyed;
  • the first_for_this_emitter bit is set if the connection object is being created and it is the first connection for the emitter passed to connect/translate;
  • the last_for_this_emitter bit is set if the connection object is being destroyed and it is the last connection for the emitter passed to connect/translate.
